# Анализ лояльности пользователей Яндекс Афиши

## Описание проекта

Яндекс Афиша — сервис, который помогает пользователям находить мероприятия в разных городах и покупать на них билеты. Афиша сотрудничает с организаторами мероприятий и билетными операторами, которые публикуют информацию о событиях и продают билеты через платформу.

Команда маркетинга поставила цель не просто привлекать новых пользователей, а удерживать их и превращать в постоянных клиентов — для этого нужно повысить долю пользователей, которые после первой покупки возвращаются и совершают повторные заказы.

**Цель проекта** — провести исследовательский анализ данных и выяснить, какие пользователи с большей вероятностью возвращаются на платформу и оформляют повторные заказы.

Результаты анализа позволят бизнесу:

- быстрее выявлять перспективных клиентов и предлагать им персонализированные условия;
- точнее настраивать рекламу на аудитории с высокой вероятностью возврата;
- оптимизировать маркетинговые бюджеты;
- повысить общий уровень удержания клиентов.

Исследовательский анализ данных (EDA) — ключевой этап работы Data Scientist: на нём изучается структура и содержание данных, выявляются взаимосвязи между переменными и формируется обоснование для дальнейших решений. EDA может быть как подготовкой к построению модели, так и самостоятельной задачей — например, проверкой бизнес-гипотез или поиском закономерностей в данных.

## Структура проекта

Проект состоит из пяти последовательных этапов — от подключения к данным до публикации готовой работы.

**1. Знакомство с данными**
Подключение к базе данных SQL и первичное знакомство со структурой и содержанием данных.

**2. Подготовка к выгрузке данных**
Решение SQL-задач для подготовки запроса, который выгружает данные для дальнейшего анализа в Python.

**3. Исследовательский анализ данных**
- Выгрузка данных из БД в датафрейм pandas.
- Предобработка данных: проверка пропусков, типов данных, аномалий и выбросов.
- Формирование профиля пользователя — признаков и факторов, потенциально влияющих на повторные заказы.
- Анализ распределений данных и поиск закономерностей.
- Корреляционный анализ, включая расчёт коэффициентов `phik`, для оценки влияния признаков на количество заказов.

**4. Выводы**
Формулирование рекомендаций для бизнеса на основе результатов исследовательского анализа.

**5. Публикация проекта**
Подготовка и публикация проекта в репозитории Git.

## Стек

Python, pandas, sqlalchemy, matplotlib, seaborn, SQL, phik
